The /goal cockpit
a statusline that tells you the truth

The v2 statusline was one magenta blob — “Pursuing goal (12.5K / 50K)” — and it rendered in unrelated sessions forever. A goal run is a flight; the statusline should be the instrument panel: what is being chased, whether it is healthy, what the evaluator just decided, and when it will likely land. Two lines, state-driven colour, owned by exactly one session. Click a state.

The native /goal evaluator returns a one-line reason every turn. Surfacing it means you always know what Claude is chasing next — no scrolling the transcript.

Pulled from the native statusline JSON. The context and rate-limit gauges turn red before they bite, so a run never dies surprised.

Why this beats v2

  • 01Session-owned, not folder-owned. The segment keys on session_id and renders only if this session owns a live goal. A new shell in an old goal directory shows nothing — the v2 bug where every session inherited a stale “Goal achieved” is gone by construction.
  • 02Terminal states don’t stick. achieved shows the landing summary, then the next turn collapses the segment back to the plain row. The statusline reflects now, not session history.
  • 03The live timer is native. v2 spawned a nohup node ticker writing to /dev/tty — which Claude Code hooks no longer have. This uses statusLine.refreshInterval; the platform re-runs the script on a timer. No daemon, no leak.
  • 04Stalled ≠ failed. A no-progress turn turns the panel amber and says so. It never reads as a dead run, and the model is never shown a one-token unmet exit. Genuinely blocked work goes orange — needs input — which is the one state designed to catch a wandering eye.
  • 05Degrades cleanly. Below ~80 columns the two lines fold to one: ◎ goal · auth API · 4/7 · 12m. The reason text is the first thing dropped, never the state.

Mockup — interactive. Colours and glyphs map 1:1 to the proposed goal-statusline.sh render states. Telemetry fields (model, cost, context_window, rate_limits, session_id) are exactly the keys Claude Code v2.1.143 pipes to a statusLine command.
